1. Summary

This document mainly introduces merchant shipper how to integrate Fengqu Open API interface.

2. Business Flow

API integration with merchant shipper is mainly in order to support the development of overseas Direction Mail business, merchant shipper is very important role and take up the high responsibility in the international transportation, the specific process is as following:

  1. (1). When domestic consumers have taken a order on fengqu, Fengqu will push the order information to overseas suppliers
  2. (2). Overseas Supplier will transfer the set of goods with parcel/package manner to merchant shipper.
  3. (3). After receiving the packeage, Merchant shipper will scan the parcel and put in storage.

the appointed place by Fengqu through international trunk line transportation, then hand over them to the local receiver designated by Fengqu.

3. API Integration Flow

In view of the above business process, merchant shipper need to integrate the APIs according to the following data flow. (Figure 3.1).

  • i. Queary the pre-alert of orders: the corresponding interface twsOpenApi.getOrderList.
  • i. Scan the package and send Stock In message back: the corresponding interface twsOpenApi.sendIn.
  • i. Weight the package and send back: the corresponding interface twsOpenApi.sendWeight.
  • i. Stock out and send message back: the corresponding interface twsOpenApi.sendOut.
  • i. Send the International Route information back: the corresponding interface twsOpenApi.sendRoute.
